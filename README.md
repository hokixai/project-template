1. **Finalize Project Name:** Make a proper `[project_name]` or use the project name provided by the user.
2. **Clone & Create:** Copy all contents of this template repository.
3. **Initialize New Repo:** Create a new remote GitHub repository named `[project_name]`.
4. **Push:** Push all copied template files to the newly created GitHub repository.

### Phase 2: Agent Context Switch & Configuration
1. **Target the New Environment:** From this point forward, your target workspace for this new project is the newly created `[project_name]` repository.
2. **Link Core Rules:** You MUST update your internal agent configuration or global `AGENT.md` to establish that your primary instruction manual for this specific project is now the `.agent/AGENT.md` located **inside the newly created repository**.
3. **Context Binding:** Ensure that any future commands executed for this project will strictly follow the rules defined in the new repository's `.agent/` folder.

1. Once the new repository is fully set up, YOU MUST DELETE this README.md.
